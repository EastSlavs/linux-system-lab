# Linux Process Visibility Demo

本项目用于演示 Linux 环境下基于 procfs 的进程伪装机制与异常负载行为，供服务器运维与安全防御排查演练使用。

## 核心演示逻辑
1. 进程名伪装：通过 prctl 和覆盖 argv 内存空间，修改进程在系统中的显示名称。
2. 异常负载模拟：利用多线程占用部分 CPU 核心，模拟业务环境中的 CPU 告警场景。
3. 动态躲避逻辑：内置遍历 /proc 目录检测监控工具（如 top、ps）并主动休眠的对抗思路（当前版本默认注释，仅作代码级展示）。

## 编译与运行
依赖 C++11 及以上标准，以及 pthread 库。

` ` `bash
g++ proc_evasion_poc.cpp -o proc_demo -pthread
./proc_demo &
` ` `

## 运维排查思路演示
当系统出现异常资源占用，但进程名称具有迷惑性时，常规排查步骤如下：

1. 定位高负载进程的 PID。
2. 绕过进程名伪装，直接查询 Linux 虚拟文件系统，获取执行程序的真实绝对路径：
   ` ` `bash
   ls -l /proc/[PID]/exe
   ` ` `
3. 校验输出结果。如果 exe 软链接指向的物理路径（如 /tmp/proc_demo）与其伪装的业务身份不符，即可判定为异常进程。
